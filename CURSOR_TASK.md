## Architecture ruling (do not deviate)

The command center (apps/command-center) is the ONLY shell on the page. Today it mounts the entire Cortex Workspace shell (its own spaces bar, View/Edit mode chrome, its own design language) inside a panel — a shell inside a shell. Dissolve that:

1. **Consume the freshly published packages:** bump @hauska/cortex-client, @hauska/tile-shell, @hauska/cortex-tiles, @hauska/document-viewer, @hauska/design-tokens → `@empressaio/*` from npm (design-tokens 0.1.0, tile-shell 0.1.0, cortex-client 0.1.0, cortex-tiles 0.1.1, document-viewer 0.1.0 — all live). `@hauska/map-renderer` stays as-is (different repo, unrenamed).
2. **The command center's own left nav owns the spaces.** The existing nav sections (COMMAND CENTER / CORTEX WORKSPACE / SUBSTRATE / ENGINES / GOVERNANCE) keep their structure; the CORTEX WORKSPACE section's entries (Plan Review, Site Analysis, Property Intel, Design Accelerator, + Print View) each open a native panel view that renders that space's tiles DIRECTLY — no inner spaces bar, no inner View/Edit chrome, no "Cortex Workspace" title row, no nested shell. Tile add/remove/layout editing can live behind a single compact "Edit layout" affordance styled like the command center's own controls; drop the inner mode toggles.
3. **One design language: the command center's.** Tiles from @empressaio packages must render in the command-center dark theme — pass the design tokens down (the packages consume @empressaio/design-tokens; theme/override at the composition root, do NOT fork the tile components). Panel chrome (borders, headers, status chips) uses the command center's existing panel styling — the same visual family as the LIVE/STUB chips and the right-rail state legend. The right-rail STATE LEGEND is load-bearing (it renders the four structural commitments) — it stays, unchanged, visible across workspace views.
4. **Derive, don't fork, the tile registry:** keep deriving panels from TILE_CAPABILITIES (the parity mechanism from PR #9) so capability drift stays structurally impossible. Status-aware stubs keep honest not-built labels.
5. **Testability — the operator's other complaint.** Every LIVE tile must actually function through the same-origin proxy against prod. Add at least a smoke-level component test for the space-panel composition (renders a space, tiles mount, no nested-shell elements present — assert the inner spaces-bar/mode-chrome DOM is GONE).
